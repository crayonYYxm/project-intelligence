Core Capabilities

它不只是"画图"，而是让 Agent 有项目记忆。

GitNexus 负责符号级调用链，Understand-Anything 负责架构上下文，Project Intelligence 把它们和项目规范、质量检查、工作流串成一个可复用的日常底座。

Init Sequence

init 不是装完就结束，而是把项目智能真正引导起来。

安装后第一次初始化怎么跑、图谱工具怎么判断、哪些步骤要停在 agent 里继续——这些边界讲清楚了，后面才顺。

1

先看图谱工具状态

执行 graph-tools --json，由 agent 用中文向用户确认是否准备缺失工具。

2

再跑主初始化

默认 project-intel init。用户明确同意后才用 --setup-missing。

3

按工具状态分支

已安装的自动分析；未准备好的进入选择流。GitNexus 走 npx gitnexus analyze，Understand-Anything 按 Codex/Claude Code 分别安装、启用并验证。

4

落盘后进入日常工作流

.project-intel/ 生成后，spec、plan、debug、review 都有固定上下文入口。

graph-tools --json 
$ project-intel graph-tools --json
[
  { "tool": "GitNexus", "state": "installable" },
  { "tool": "Understand-Anything", "state": "installable",
    "installOptions": ["codex", "claude"] }
]

agent next step:
1. 中文提示用户：全部 / 组合 / 跳过
2. 先执行 shell 可运行的 setup
3. Claude Code 安装后先验证 plugin list
4. 验证成功再提示 /reload-plugins、/understand 与 refresh
关键点

init 的价值不是输出一句"初始化成功"，而是把后续 agent 需要依赖的项目知识真正沉淀下来。

Get Started

安装只是入口，真正开始用是跑完第一次 init。

将仓库添加为 marketplace 后即可安装。安装完成，切到目标仓库跑 init 即可。

🟣 Claude Code

添加 marketplace 仓库：

/plugin marketplace add crayonYYxm/project-intelligence

安装插件：

/plugin install project-intelligence@project-intelligence

🟢 Codex CLI

添加 marketplace 仓库：

codex plugin marketplace add crayonYYxm/project-intelligence

安装插件：

codex plugin install project-intelligence@project-intelligence
下一步

安装完成后，cd 到目标仓库，执行 project-intel init 即可。init 会自动检测图谱工具状态并引导后续步骤。
