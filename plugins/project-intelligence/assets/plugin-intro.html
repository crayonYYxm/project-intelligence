What It Covers

不止 `init`。它覆盖项目从需求到维护的完整链路。

核心目标是减少 Agent 猜测：做需求先查复用，写方案先查影响，排 bug 先建上下文，review 先看规范和变更风险。

01 / lifecycle

需求脑暴、spec、plan

从一句需求开始，先整理目标、边界、影响范围和实现计划，再进入编码。

02 / standards

项目规范沉淀

把 hard、preferred、inferred、candidate 四级规则写入 `.project-intel/standards/`。

03 / reuse

组件与公共能力复用

扫描前端组件、Hooks、API、路由、状态管理和重复候选，做需求前先查可复用资产。

04 / graph

GitNexus + Understand-Anything

GitNexus 负责符号级调用和影响分析，Understand-Anything 负责架构、模块和领域流。

05 / debug

系统化排错

debug 前生成上下文，要求复现、读错误、查最近变更、追调用链，再提出单一假设。

06 / quality

质量与冗余检查

识别项目已有 lint、type-check、format、style 工具，并输出重复 UI、逻辑和样式候选。

Natural Language Usage

用户不用记 CLI。日常用自然语言触发。

CLI 是底层工具；真正使用时，Claude Code / Codex 通过 skills 自动读 `.project-intel`、图谱摘要、规范和报告。

A

初始化项目规范

检测工具、生成规范、扫描知识库，并维护 AGENTS.md / CLAUDE.md 项目入口。

B

做需求前先查上下文

读取组件、服务、API、业务流、图谱影响面和复用候选。

C

实现后维护项目事实

运行 check、refresh、maintenance report，让知识库跟上代码变化。

初始化这个项目规范 执行 init，检查 GitNexus、Understand-Anything、质量工具和项目结构。
帮我做这个需求 触发 project-task，先读规范、复用资产和图谱上下文，再实施。
帮我写 spec / plan 输出需求说明、影响关系、技术计划和可执行 checklist。
查询这个 bug 触发 project-debug，使用 systematic-debugging 流程定位根因。
帮我 review 代码 读取 diff、规范、质量结果和图谱影响面，按严重度输出 findings。
图谱跑完了 立即触发 project-refresh，刷新 `.project-intel` 中的图谱摘要和项目知识。
Graph & Setup

图谱增强是可选的，但闭环是自动接上的。

GitNexus 能在 shell 里直接分析；Understand-Anything 在 Claude Code 里需要 `/reload-plugins` 和 `/understand`。图谱完成后，插件会引导或直接执行刷新。

2. 仅 GitNexus：运行 node .gitnexus/run.cjs analyze
3. 仅 Understand-Anything：安装到 Claude Code
4. 跳过图谱增强：直接初始化 .project-intel/

  /plugin marketplace add Egonex-AI/Understand-Anything

  /understand . --language zh

after graph:
  /project-refresh
  or fallback: project-intel refresh
边界清晰

CLI 不会假装能注入 Claude Code slash command。它会完成能在 shell 里做的安装、启用、验证；图谱完成后通过 project-refresh 把结果写回 `.project-intel`。

这些 skills 让 Agent 在不同任务中读同一套项目事实，而不是每次从零理解仓库。

Repository Artifacts

所有项目事实都沉淀在仓库内。

`.project-intel/` 是可提交、可审查、可刷新、可被 Claude Code 和 Codex 共用的事实层。缓存和中间产物默认不提交。

metadata

manifest.json

技术栈、入口、图谱来源、刷新状态和 commit。

rules

standards/*.md

组件、Hook、API、后端分层、权限、错误码和测试规范。

knowledge

knowledge/*.json

组件、Hooks、API、服务、DTO、路由、配置和业务流程。

graph

graph/project-graph.json

统一引用 GitNexus 和 Understand-Anything 的摘要。

reports

reports/*.md

初始化、质量、冗余、影响分析、review 和 debug 报告。

lifecycle

specs/ plans/ maintenance/

需求、计划和任务后维护记录，形成可追踪时间线。

Install

装上插件，切到仓库，开始沉淀项目智能。

安装只是入口。第一次 `init` 负责把项目规范、知识库、图谱状态和 AGENTS.md / CLAUDE.md 项目规则接起来，后续需求和 review 都会复用这些事实。
